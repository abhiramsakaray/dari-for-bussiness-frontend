## Token Decimals Handling

### BSC Decimal Exception

**IMPORTANT:** BSC uses 18-decimal stablecoins, not the standard 6 decimals.

| Chain | USDC Decimals | USDT Decimals |
|-------|---------------|---------------|
| Ethereum | 6 | 6 |
| Polygon | 6 | 6 |
| Base | 6 | - |
| **BSC** | **18** | **18** |
| Avalanche | 6 | 6 |
| Tron | 6 | 6 |

### Current Implementation Status

✅ **No frontend changes needed** - The frontend does NOT handle token decimals directly:
- All amount conversions are handled by the backend
- The frontend only displays pre-formatted amounts from the API
- QR codes are generated by the backend
- No Web3 amount conversions in the frontend (except for Web3 subscriptions)

✅ **Web3 Subscriptions** - Already handles decimals correctly:
- Uses `tokenDecimals` variable that can be set per token
- Has `parseUnits()` and `formatUnits()` functions that accept decimal parameter
- Currently hardcoded to 6 decimals for Polygon USDC/USDT
- **Action Required:** If BSC subscriptions are added, update token metadata to use 18 decimals
